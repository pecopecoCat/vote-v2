# VOTE v2 テスト計画・チェックリスト

手動テスト用の全体チェックリストです。リリース前・機能追加後に使ってください。

**スプレッドシート版（CSV）**: [spreadsheet/](./spreadsheet/) フォルダを Google スプレッドシートにインポートしてください。手順は [spreadsheet/README.md](./spreadsheet/README.md) を参照。

## 使い方

1. **環境**と**アカウント**を下表のとおり準備する
2. 各項目を **OK / NG / 保留** で記録する（スプレッドシートにコピーしても可）
3. NG は「再現手順・端末・スクショ・期待との差」をメモする
4. **P0** は必ず全端末で実施、**P1** は主要フロー、**P2** は余力時

### 記号

| 記号 | 意味 |
|------|------|
| P0 | リリースブロッカー級（投票・コメント・ログイン・データ消失） |
| P1 | 主要機能 |
| P2 | 周辺・UI・エッジケース |

## テスト環境

### 推奨端末（最低セット）

| # | 端末 | ブラウザ | 理由 |
|---|------|----------|------|
| A | iPhone 実機 or Simulator | **Safari** | タブ中央揃え・キーボード・スクロール差分 |
| B | Mac / Windows | **Chrome** | 開発時の基準 |
| C | 同上 | **Firefox**（任意） | レイアウト差分の確認 |

### データモード（2パターン両方見る）

| モード | 条件 | 確認すること |
|--------|------|----------------|
| **ローカルのみ** | `KV_REST_API_*` 未設定で `npm run dev` | localStorage のみ・単端末 |
| **KV 共有** | Vercel 本番 or `.env.local` に KV 設定 | 投票・コメント・作成VOTE が他端末と共有される |

### テスト用アカウント

| 種別 | 操作 |
|------|------|
| ゲスト | ログインせず利用 |
| デモ user1〜user6 | `/profile/login` から選択ログイン |
| 2ユーザー同時 | user1 と user2 を別ブラウザ／シークレットで（KV・重複ログイン確認） |

### 画面サイズ

- **375px 幅**（iPhone 標準）を基準に確認
- 可能なら 320px（小型）・414px もサンプル確認
